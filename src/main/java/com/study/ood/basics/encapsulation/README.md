# Encapsulation

## Data Hiding
Data hiding is a fundamental concept in object-oriented programming (OOP) that involves bundling data (attributes) and the methods that operate on that data into a single unit called a class.

The goal is to protect the data from outside access and modify it only through the methods of the class.

### Component of data hiding
Data handing can be divided into two components: 
1. Encapsulation
2. Abstraction

## Encapsulation
Encapsulation is the technique of hiding the internal implementation details of an object from the outside world.

### Why encapsulation?
There are several reasons to use encapsulation:
1. Data protection: Encapsulation helps protect the internal state of an object from outside access. 
2. Modularity: Encapsulation helps in modular design by encapsulating related data and methods into a single unit. 
3. Flexibility: By properly encapsulating the data, we can change the internal implementation details of a class without affecting the external code that uses it. 
4. Maintainability: Encapsulation helps in maintaining the code by providing a clear interface for the object. 

### how to achieve encapsulation
We can achieve encapsulation in java by using the following techniques:
1. Data hiding: Make all data members of the class private.
2. Getters and setters: Provide public getter and setter methods for the data members.
