# About Object-Oriented Programming (OOP)

## 1. Definition
Object-oriented programming is a programming style, not a tool, so despite being old, it's paradigm is still widely used.
This programming style involves dividing a program into objects, each of which has its own data and methods.
Every object has it's own unique set of properties (data) and methods (functions). Each object can also interact with other objects to perform complex tasks.

## 2. Building blocks of OOP
The following are the essential concepts of OOP:
- Attributes
- Methods
- Classes
- Objects

### 2.1. Attributes 
Attributes are variables that define the state of an object. For example, if we have a class called Dog, its attributes could be its name, breed, and age.

### 2.2. Methods
Methods are functions that define the behavior of an object. For example, if we have a class called Dog, its methods could be its bark, run, and sleep.

### 2.3. Classes
Classes are the blueprints for creating objects. For example, if we have a class called Dog, it acts as a template defining what attributes (name, breed) and methods (bark, run) a dog should have.

### 2.4. Objects
Objects are instances of a class. For example, if we have a class called Dog, an object could be a specific dog like a Poodle named "Buddy" or a Bulldog named "Max".

## 3. Principles of OOP
There are four fundamental principles of OOP:
- Abstraction: Hiding complex implementation details and only showing essential features.
- Encapsulation: Restricting direct access to attributes using private access modifiers, and providing public getter/setter methods.
- Inheritance: Acquiring the properties and behaviors of another class.
- Polymorphism: Allowing a class to have many forms.
